# 1 概要设计
## 1.1 设计目标
1. 遵循需求分析文档，实现**全生命周期飞行计划管理**
2. 采用**B/S架构、前后端分离、模块化分层**设计
3. 满足**100并发、高可用、高安全、易扩展**要求
4. 严格匹配三级审批、实时监控、数据分析、系统管理核心业务
5. 支持飞控/地图接口对接，满足异常预警与数据归档

## 1.2 总体架构设计
系统采用**四层架构+接口层**，高内聚低耦合，便于开发、测试、扩展。

### 1.2.1 前端展示层
- 技术：Vue3 + Element Plus + Axios
- 功能：页面渲染、表单校验、权限控制、地图展示、实时数据刷新
- 适配：Chrome 90+、Firefox 88+、Edge 90+、Safari 14+

### 1.2.2 后端服务层
- 技术：Spring Boot 2.7.x
- 核心模块：用户认证、计划管理、审批流、飞行监控、数据分析、系统配置
- 安全：Spring Security + JWT Token + RBAC权限

### 1.2.3 数据持久层
- 数据库：MySQL 8.0
- 持久化：MyBatis-Plus
- 缓存：Redis（可选，用于高频数据、接口缓存）

### 1.2.4 接口适配层
- 内部接口：RESTful API，JSON格式
- 外部接口：飞控接口、地图接口（HTTP/HTTPS）
- 统一：请求重试、熔断、日志、异常处理

## 1.3 功能模块结构设计
系统共**5大核心模块**，完全对齐需求：

1. **飞行计划管理模块**
   - 创建/编辑/删除/查询/提交计划
   - 计划状态流转、附件上传、区域校验

2. **审批管理模块**
   - 三级审批流：初审→复审→终审
   - 待办提醒、超时提醒、审批意见、流程跟踪

3. **飞行监控模块**
   - 实时数据采集、地图轨迹、偏航/低电量/超区预警
   - 历史轨迹回放、预警记录查询

4. **数据分析模块**
   - 计划/审批/飞行多维度统计
   - 可视化图表、日报/月报/年报、导出Excel/PDF

5. **系统管理模块**
   - 用户/角色/权限管理
   - 空域/无人机/任务类型维护
   - 接口配置、日志管理、数据备份恢复

## 1.4 系统流程设计
### 1.4.1 主业务流程
用户登录 → 创建计划 → 提交 → 三级审批 → 审批通过 → 飞行执行 → 实时监控 → 任务完成 → 数据分析 → 自动归档

### 1.4.2 审批流程
提交 → 初审 → 复审 → 终审 → 通过/驳回
驳回 → 用户修改 → 重新提交

### 1.4.3 异常流程
- 审批驳回：通知→修改→重提
- 飞行异常：预警→人工处理→记录→归档

## 1.5 接口设计概要
### 1.5.1 内部接口规范
- 协议：HTTP/HTTPS
- 风格：RESTful
- 格式：JSON
- 鉴权：Header携带JWT Token
- 统一返回：code、msg、data

### 1.5.2 外部接口
1. **飞控接口**
   - 频率：5秒/次
   - 数据：经纬度、高度、速度、电量、信号、模式
2. **地图接口**
   - 底图、坐标解析、区域标注、轨迹绘制

## 1.6 安全设计
1. **身份认证**：账号密码+JWT，5次错误锁定15分钟
2. **权限控制**：RBAC三角色（用户/审批员/管理员）
3. **数据安全**：密码MD5加密、手机号脱敏、HTTPS传输
4. **防攻击**：SQL注入过滤、XSS过滤、CSRF防御
5. **操作安全**：核心操作二次确认、全链路日志

## 1.7 性能设计
- 页面加载≤2s
- 接口响应≤3s
- 支持100并发
- 飞行数据刷新≤5s
- 报表生成≤1~3s

# 2 界面设计
## 2.1 设计原则
- 简洁统一、流程清晰
- 权限可见、操作可追溯
- 地图可视化、预警醒目
- 适配主流浏览器

## 2.2 公共界面
### 2.2.1 登录页
- 账号、密码、验证码
- 记住我、忘记密码（管理员重置）
- 角色自动识别跳转

### 2.2.2 主框架页
- 左侧：菜单导航（按权限显示）
- 顶部：用户信息、消息中心、退出
- 主体：内容区
- 底部：版本、版权

## 2.3 普通用户界面
### 2.3.1 飞行计划列表
- 查询：计划名、状态、时间、任务类型
- 操作：查看、编辑、删除、提交、跟踪
- 表格：编号、名称、时间、区域、高度、状态、创建时间

### 2.3.2 创建/编辑飞行计划
- 基础信息：计划名、时间、区域（地图选点）、高度、任务类型、无人机型号
- 人员信息：操作员、电话
- 描述、附件上传（PDF/Word/Excel ≤10M）
- 保存草稿 / 提交审批

### 2.3.3 审批状态查看
- 流程进度条：初审→复审→终审
- 每级：审批人、时间、意见、结果

### 2.3.4 飞行监控页
- 地图：实时位置、预设轨迹、实际轨迹
- 数据面板：经纬度、高度、速度、电量、信号
- 预警区：红色高亮、弹窗提示

### 2.3.5 个人报表
- 个人飞行统计、任务统计、异常统计
- 图表：柱状图、饼图
- 导出Excel/PDF

## 2.4 审批人员界面
### 2.4.1 待审批列表
- 标红：超时未处理
- 字段：计划编号、名称、申请人、时间、区域、状态
- 操作：查看详情、审批（通过/驳回+意见）

### 2.4.2 已办审批
- 按时间/结果/节点查询
- 查看审批历史、流程轨迹

### 2.4.3 审批统计
- 待办数、已办数、及时率、驳回率
- 按日/月统计

## 2.5 管理员界面
### 2.5.1 用户管理
- 增删改查、角色分配、启用/禁用、密码重置
- 查询：账号、姓名、角色、状态

### 2.5.2 权限配置
- 角色：普通用户、审批员、管理员
- 菜单权限、按钮权限、数据权限

### 2.5.3 系统配置
- 审批超时时间、预警阈值
- 飞控接口地址、密钥
- 地图接口配置
- 空域维护：适飞区/限飞区/禁飞区
- 无人机型号、任务类型维护

### 2.5.4 日志管理
- 登录日志、操作日志、系统日志
- 查询、导出、不可删除

### 2.5.5 数据备份与恢复
- 手动备份、自动备份周期配置
- 备份文件列表、恢复（二次确认）

### 2.5.6 全局数据大屏
- 计划总数、通过率、飞行次数、异常率
- 实时监控概览、审批效率
- 图表可视化

## 2.6 界面交互规则
1. 新增/编辑：表单实时校验，错误标红提示
2. 提交/审批：二次确认
3. 预警：顶部弹窗+声音+消息中心
4. 列表：支持分页、排序、模糊查询
5. 地图：支持缩放、拖拽、轨迹回放

# 3 数据库设计
## 3.1 设计原则
- 符合需求分析数据实体
- 三范式为主，适度冗余
- 主键自增、外键关联、索引优化
- 字段非空约束、唯一约束、默认值
- 支持大数据量、快速查询

## 3.2 数据库命名规范
- 库名：uav_flight_system
- 表名：小写+下划线，如user、flight_plan
- 字段：小写+下划线，如user_id、plan_no
- 主键：id
- 外键：xxx_id
- 时间：create_time、update_time

## 3.3 数据库表结构（完全对齐需求）

|config_key|varchar(64)|NN UK|配置键|

|remark|varchar(100)||备注|

**用途**：审批超时、预警阈值、接口地址、备份周期等

## 3.4 索引设计
- user：user_account(UK)
- flight_plan：plan_no(UK)、user_id、status、flight_start
- approval_record：plan_id、approver_id
- flight_data：plan_id、collect_time
- warning_record：plan_id、warning_time
- system_log：log_type、user_id、create_time

## 3.5 数据安全与归档
1. **加密**：密码MD5
2. **脱敏**：手机号138****1234
3. **事务**：提交计划、审批、数据写入使用事务
4. **备份**：每日自动备份，保留6个月
5. **归档**：已执行→已归档，只读不可改
